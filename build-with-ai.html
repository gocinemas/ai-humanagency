🛠️ Builder's Guide

Build with AI

You have an idea. You don't know where to start. This guide answers the three questions that matter: What model? How to test? How to turn it into an app?

Step 1: What Model Do You Need?
What is your idea about?

✍️ Text Model

Use: Claude or GPT-4 for anything involving writing, analysis, reasoning, or code.

Examples: Customer service chatbots, document analysis, email drafting, code generation.

Start with: Claude.ai or ChatGPT free tier. Test your idea in 5 minutes.

🖼️ Image Model

Use: Claude Vision, GPT-4V for reading/understanding images. DALL-E or Midjourney for generating images.

Examples: Garden planning (scan photo, get recommendations), document scanning, visual inspection, design generation.

Start with: Claude.ai (upload a photo and ask a question). Free. Instant.

🎤 Audio Model

Use: Whisper for transcription. GPT-4 for analysis of transcribed text.

Examples: Meeting transcription, voice note analysis, podcast summarisation.

Start with: OpenAI's Whisper API or use a tool like Otter.ai that wraps it.

🔀 Multimodal Model

Use: Claude Vision + text, or GPT-4o (handles text, images, audio together).

Examples: Scan a garden photo, get text recommendations, export as a plan.

Start with: Claude.ai. Upload image, ask your question, get text back.

Step 2: Test Your Idea (This Week)

Garden Scanning App Example

The idea: Scan your garden → AI suggests where to plant, play with layout.

Test in 30 minutes:

  1. Take a photo of a garden (yours or Google Images)
  2. Go to Claude.ai, upload the photo
  3. Ask: "I want to plan what to grow here. What are the zones? What would you recommend? Draw a simple plan."
  4. See what you get back. Is it useful? Does it answer your question?
  5. If yes → keep going. If no → rephrase the question and try again.

Key insight: You don't need to build anything yet. You're just proving the idea works. If Claude can't handle it, neither can your app. If Claude nails it, you know the idea is sound.

Step 3: Turn It Into an App
What type of app do you want?

🌐 Web App

Tech stack: React or Vue (frontend) + Node.js (backend) + Claude API.

Timeline: 2-4 weeks for MVP (minimum viable product).

Cost: Hosting (~£5-20/month) + API calls (~£0.01-1 per request).

Start: Learn React basics or use a no-code tool like Bubble or Webflow with AI integrations.

📱 Mobile App

Tech stack: React Native or Flutter + Claude API.

Timeline: 4-8 weeks for MVP.

Cost: App store fees (£99 Apple, free Google) + API calls.

Start: Prototype with Expo (React Native) or Flutter. Both have good documentation.

⚙️ Web + Mobile

Tech stack: React (web) + React Native (mobile) sharing code. Backend: Node.js + Claude API.

Timeline: 8-12 weeks.

Cost: Higher upfront (more development), lower per-user cost.

Start: Build web first. Once it's working, port to mobile using React Native.

The Real Path Forward
  1. Test with Claude.ai this week. Prove the idea works. (30 mins)
  2. Build a simple web prototype. Basic interface, API calls to Claude. (2 weeks)
  3. Get real users. Let 5-10 people test it. See what breaks. (1 week)
  4. Iterate based on feedback. Fix the broken bits. (1-2 weeks)
  5. Only then decide: Mobile app? Better hosting? Premium features?

Most builders skip steps 1-4 and jump to "build the perfect app." That's backwards. Test first. Build second. Perfect third.
